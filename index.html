Users

My Favorite Albums is a web application that allows users to explore and analyze one person's album listening history through interactive data visualizations.

In this guide, you will learn how to use each of the data aggregation tools within My Favorite Albums. Use the links below to jump to any tab.

Audience: End users with little to no programming experience who want to explore and analyze the owner's album collection through the My Favorite Albums web interface.

Prerequisites: Familiarity with basic web browser interaction such as clicking buttons, selecting from menus, and dragging sliders.

Starting point: The My Favorite Albums website is open in your browser.

Note: any words that are bolded have definitions in the appendix at the bottom of this page.

Use this tab to browse the owner's top-ranked album for each year within a time period you define. By adjusting the range slider, you can expand or narrow the year range, and the data table will update to show the number one album for each year in your selected range. This walkthrough will guide you through the range slider with an example.

  1. Click the Number One Albums tab. You will see a range slider and 6 albums in a data table like the image below (the range slider is set to start from 1993 to 1998 at the beginning).
  2. Drag your mouse over the right handle and slide it to 2010. If done correctly, you will see the last album in the data table has the year 2010, as shown below.
  3. Drag your mouse over the left handle and slide it to 2000. If done correctly, you will now see the first album in the data table contains the year 2000 or later, as shown below.

Use this tab to view all of the owner's ranked albums for a single year of your choosing, displayed from highest to lowest rank. Select a year from the drop-down menu and the data table will show every album the owner ranked that year, along with each album's ranking and artist.

  1. Click the Top Albums by Year tab.
  2. Click the Choose a year drop-down menu and select the year you want to explore.
  3. Click Submit. If done correctly, a data table appears showing all of the owner's ranked albums for that year, sorted from highest to lowest ranking. The table contains the fields Ranking, Album, and Artist, as shown below.

Use this tab to explore every album the owner has ranked for a specific artist. Select an artist from the drop-down menu to see their full list of ranked albums along with each album's release year and rating. The tab also displays the artist's total ranked album count and average rating below the data table.

  1. Click the Artist's Albums tab.
  2. Click the Choose a band or artist drop-down menu and select an artist. Artists are listed in alphabetical order.
  3. Click Submit. If done correctly, a data table appears showing that artist's ranked albums with the fields Album, Year, and Rating. Below the table, the artist's total ranked album count and average rating are displayed, as shown below.

Note: if the table does not appear after clicking Submit, re-select the artist from the drop-down menu and click Submit again.

Use this tab to discover the owner's highest-rated artists, filtered by a minimum number of ranked albums. Set the minimum album count to focus on artists the owner has listened to most, and optionally exclude EPs and live albums from that count. The results display up to 15 artists sorted by average rating in descending order.

  1. Click the Favorite Artists tab.
  2. Click the Minimum number of albums drop-down menu and select the minimum number of albums an artist must have to appear in the results.
  3. Check or uncheck the Exclude EPs and Live Albums checkbox depending on whether you want those release types included in the album count.
  4. Click Submit. If done correctly, a data table appears showing up to 15 artists with the fields Artist, Rating, and number of albums, sorted by average rating from highest to lowest, as shown below.

Use this tab to generate a comparative line chart that plots two artists' average album ratings side by side across different years. Select two artists from the drop-down menus and the chart updates automatically, making it easy to see how each artist's ratings compare to one another over time.

  1. Click the Artist Comparison tab.
  2. Click the First band or artist drop-down menu and select your first artist.
  3. Click the Second band or artist drop-down menu and select your second artist. If done correctly, the comparative line chart generates automatically — no Submit button is needed.
  4. Read the chart: the horizontal axis shows the year, the vertical axis shows average rating, the first artist's line is red, and the second artist's line is blue. Each point represents that artist's average rating across all albums the owner ranked in that year.

Note: an artist will only appear on the chart for years in which the owner has at least one ranked album.

Use this tab to find highly rated albums that the owner does not yet own on vinyl. Set a minimum rating threshold from the drop-down menu to filter the results, and the data table will display all qualifying albums sorted by rating — making it easy to identify top candidates for the vinyl collection.

  1. Click the Vinyl tab.
  2. Click the Select which albums to display drop-down menu and select a minimum rating threshold.
  3. Click Submit. If done correctly, a data table appears listing all qualifying albums with the fields Album, Artist, and Rating, sorted by rating from highest to lowest and then alphabetically by album name, as shown below.
  • Range Slider: A user interface component that allows you to select a range of values from a set of numbers.
  • Handle: The draggable element on a range slider that users interact with to select a value or range of values. It moves along the slider's track and visually indicates the current selected value(s). In a dual-range slider, there are typically two handles—one for the minimum value and one for the maximum value—allowing users to define a specific range.
  • Drop-Down Menu: A graphical user interface element that allows users to choose from a list of options that appears when they click on or hover over a button or other control.
  • Comparative Line Chart: A visual representation of data used to compare and contrast two or more data sets.
  • Aggregate: To filter and combine data based on criteria you select.

This is the CSV file that all data aggregation functions operate on. Follow these steps to create or update it with your own album data. Getting this file structured correctly is required before the app will load and display any data.

  1. Open a spreadsheet application such as Excel or Google Sheets and create a new blank file.
  2. In the first row, enter the following column headers in this exact order: 
    Year, Ranking, Album, Artist, Rating, Vinyl, EP, Live
  3. Enter each album as its own row beneath the headers. For the Vinyl, EP, and Live columns, enter TRUE or FALSE in all caps — R reads these as logical values, and lowercase entries will cause filtering errors.
  4. Save the file as a CSV. In Excel, choose File > Save As and select CSV (Comma delimited). In Google Sheets, choose File > Download > Comma-separated values (.csv).
  5. Name the file album-rankings.csv and place it in the data/ folder inside the project directory.
  6. Open RStudio, set your working directory to the project folder via Session > Set Working Directory > Choose Directory, and run the app with shiny::runApp(). If the Home tab displays a total album count and artist count, the file loaded correctly.

Note: the Artist field must be spelled consistently across all rows. Inconsistent capitalization (e.g., "Radiohead" vs. "radiohead") will cause that artist to appear as multiple separate entries throughout the app.

All data functions can be called directly in the RStudio console without running the full Shiny app. This is useful for testing individual functions, debugging output, or exploring the data outside of the web interface. Follow these steps to source and call any function from the console.

  1. In the RStudio console, load dplyr and read the dataset: 
    library(dplyr)
album_data <- read.csv("data/album-rankings.csv")
  2. Source the file containing the function you want to call: 
    source("albums_by_band.R")
  3. Call the function, passing album_data as the first argument. For example: 
    albums_by_bands(album_data, "Radiohead")
    A data frame appears in the console with the columns Album, Year, and Rating.
  4. For band_album_comparison_chart(), also load ggplot2 before calling: 
    library(ggplot2)
source("compare_bands.R")
band_album_comparison_chart(album_data, "Radiohead", "Arcade Fire")
    The chart renders in the RStudio Plots pane.

Note: artist name arguments are case-sensitive and must match the dataset exactly. Run unique(album_data$Artist) to see all valid artist names.
